## The ORDER BY Method – How It Works

You tell the database: "Sort the results by column number X"

If column X doesn't exist, the database throws an error.

## Simple Step-by-Step

**Start with column 1:**
```
' ORDER BY 1 --
```
No error → column 1 exists

**Try column 2:**
```
' ORDER BY 2 --
```
No error → column 2 exists

**Try column 3:**
```
' ORDER BY 3 --
```
No error → column 3 exists

**Try column 4:**
```
' ORDER BY 4 --
```
ERROR → column 4 does NOT exist

**Conclusion:** The original query returns **3 columns**

## What the Error Looks Like

The database error might be:
- `The ORDER BY position number 4 is out of range`
- `Unknown column '4' in 'order clause'`
- `Column index out of range`

Or the application might just show:
- A blank page
- A generic error message
- Missing data

## Why This Works

You don't need to know column names. The database understands `ORDER BY 3` means "the third column in the result set."

## One Sentence Summary

Increase the number after `ORDER BY` until you get an error – the highest working number is your column count.
