You've covered a very clear example of **authentication bypass**. Let me explain the simple concept.

## The Core Idea

The application checks both username **AND** password. You want to remove the password check entirely.

## How It Works

**Normal login query:**
```sql
SELECT * FROM users WHERE username = 'wiener' AND password = 'bluecheese'
```

**You send:**
- Username: `administrator'--`
- Password: (anything, it doesn't matter)

**Resulting query:**
```sql
SELECT * FROM users WHERE username = 'administrator'--' AND password = ''
```

The `--` comments out the rest. So the database only checks:
```sql
WHERE username = 'administrator'
```

The password condition is completely ignored.

## What Happens

The database finds the user `administrator` in the table. It doesn't care what password you typed. You're logged in as that user.

## Simple Example

| You type | What SQL sees |
|----------|---------------|
| Username: `admin'--` | `WHERE username = 'admin'` |
| Password: `anything` | (ignored) |

## The Danger

This works for **any** username you know:
- `administrator'--`
- `admin'--`
- `root'--`

If you don't know a username, try `admin' OR 1=1 --` to log in as the first user in the database (often an admin).

## One Sentence Summary

You inject `'--` after the username to turn the password check into a comment, so the database only verifies the username exists.
